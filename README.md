# 🏠 UNILAND (대학생 맞춤형 부동산 중개 플랫폼)
"처음 자취를 시작하는 대학생들을 위한 예산/거리/편의성 기반 맞춤형 매물 추천 서비스"

기존 부동산 플랫폼의 복잡함을 걷어내고 대학생의 생활 패턴과 예산에 최적화된 주거 솔루션을 제공합니다.

프로젝트의 기획 의도, 시스템 아키텍처, 핵심 기능 시연 화면이 포함된 발표 자료입니다.

1. 개요 및 배경
수행 기간: 2025.09.27 ~ 2025.10.31 (팀 프로젝트) 
주요 타깃: 대학 주변 매물 정보가 필요한 대학생 
핵심 목표:
(1) AI 자연어 검색을 통한 직관적인 매물 탐색 
(2) 예산 계산기 및 카카오맵 기반 지도 시각화 
(3) 사용자/중개사/관리자 권한별 맞춤 기능 제공 


2. 사용 기술 (Tech Stack)
Language: Java, JavaScript, HTML5, CSS3 
Framework: Spring Boot, MyBatis 
Database: Oracle 
API: OpenAI GEMINI (자연어 검색), Kakao Map API, 국토교통부 실거래가 API 
Tool/WAS: GitHub, SQL Developer, Apache Tomcat 


3. 핵심 담당 기능 (Individual Role)
저는 백엔드 로직 구조화와 데이터 처리 효율성 향상을 목표로 공지사항 및 관리자 전반의 기능을 담당했습니다. 

✅ 관리자 페이지 및 대시보드 (Full-stack)
쿼리 최적화 및 인덱스 설계: 복잡한 테이블 조인 구조를 개선하고 인덱스를 추가하여 데이터 조회 속도를 약 70% 단축했습니다. 
AJAX 비동기 통신: 회원/매물 관리 시 페이지 새로고침 없이 데이터를 갱신하여 사용자 대기 시간을 감소시켰습니다. 
권한 기반 관리: 중개사 회원가입 승인 및 전체 매물/문의 내역 관리 기능을 Spring MVC 패턴으로 구현했습니다. 

✅ 공지사항 및 사용자 문의 (Full-stack)
CRUD 구현: MyBatis를 이용한 효율적인 SQL 매핑으로 공지사항 등록/수정/삭제/조회 기능을 개발했습니다. 
동적 UI: JSP와 JavaScript를 활용해 실시간 반영되는 사용자 중심 인터페이스를 구축했습니다. 


4. 트러블 슈팅 (Problem Solving)
문제: 더미 데이터 증가 및 복잡한 Join으로 인해 관리자 페이지 로딩이 5초 이상 지연됨. 
분석: 비효율적인 쿼리 구조 및 인덱스 부재 확인.
해결: SQL 쿼리 최적화, 인덱스 추가, AJAX 비동기 통신 적용. 
결과: 응답 속도를 70% 이상 개선하여 원활한 관리 서비스 제공 가능. 


5. 협업 및 성장 경험
Git 전략: 브랜치 전략(Branch Strategy)과 PR(Pull Request) 리뷰 프로세스를 주도하여 코드 충돌을 최소화했습니다. 
코드 품질: MyBatis 매퍼 네이밍 규칙 정의 및 공통 ResultMap 설계를 통해 유지보수성을 높였습니다. 


소감: "기술로 사용자의 불편을 해결하는 과정에서 개발자로서의 가치를 느꼈으며, 문제의 본질을 분석하고 성능을 개선하는 역량을 길렀습니다."
